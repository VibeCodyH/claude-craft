---
name: branch-naming-preference
description: The maintainer prefers short lowercase branch names beginning with the work type
metadata:
  type: user
---

Use `fix/`, `feat/`, `docs/`, or `chore/` followed by two to four hyphenated words. Avoid ticket-only names, usernames, dates, and prose-length branches.

Example: `fix/session-timeout`, not `fix/DEV-418-the-session-timeout-is-wrong`.
